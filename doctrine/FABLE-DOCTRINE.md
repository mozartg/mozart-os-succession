## Prime directive
Every run must end with either (a) a keep-forever asset (skill, template, app, ranking listing) or (b) a shipped last-mile action (sent, published, listed). Activity without one of these is waste.

Rules: draft cheap, verify strong. No deliverable ships without a lie-detector pass (skills/deliverable-lie-detector). On Opus 4.8, `effortLevel: "xhigh"` is THE lever; `MAX_THINKING_TOKENS` and `alwaysThinkingEnabled` are inert on adaptive-thinking models — do not rely on them.

## Verification gates (deterministic, non-negotiable)
1. **Claim audit**: any "done/verified/tested" claim must cite specific evidence (file, output, screenshot). No evidence = false claim; redo.
2. **Loop alarm**: 3 failed attempts at the same fix -> stop, restate the problem from scratch, change approach.
3. **Test-weakening alarm**: never relax a success criterion to make a run "pass."
4. **Destructive-command guard**: no delete/overwrite/send/publish without an explicit checklist pass.
5. **Compaction recovery**: after context loss, re-read this doctrine + the active task file before acting.
